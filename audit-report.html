セキュリティ対応を最優先に並べています。いずれも着手前に方針をご相談し、承認をいただいてから実施します。

  1. 【最優先・残作業】リポジトリの非公開化(あなたの操作・約2分)
    履歴清掃は実施済み。仕上げとして、GitHub のリポジトリページ → Settings → 最下部の Danger Zone →「Change repository visibility」→ Private に変更。徹底するなら、あわせて GitHub サポートに古い版のキャッシュ消去を依頼(任意)。
  2. 【高】作りかけの 2 ブランチの行き先を決める
    消防専門家エージェントチーム(PR #1)と音声メモ→カレンダーアプリを、それぞれ「採用してマージ」か「削除」に振り分ける。宙に浮いた作りかけをなくすのが目的。
  3. 【高】指揮支援 v2 を正しい形で保存し直す
    claude.ai の画面まるごとではなく、アプリ本体だけの HTML として書き出し直す。単体で開いて動く形になり、個人情報の埋め込みもなくなる(元データはお渡ししたバックアップにあり)。あわせて README に説明を書く。
  4. 【中】kobe-fire-team スキルの中身点検
    スキル本文を見せてもらい、個人情報の書き込み・PR #1 のエージェントチームとの役割重複・古くなった手順がないかを点検する。
  5. 【中】GAS・Drive スクリプトの棚卸し
    場所を教えてもらうか、次回 Drive の読み取りを承認してもらえれば、同じ形式で監査して本レポートに追記する。
  6. 【低】未使用の接続・プラグインの整理
    Canva・Microsoft 365 など使っていない接続を外し、Claude に渡す権限を「使うものだけ」にする。
  7. 【低】CLAUDE.md 整備 と 定期チェックの自動化
    申し送り書(CLAUDE.md)を作り、あわせて「週 1 回、リポジトリに秘密情報が混ざっていないか自動チェック」などの定期実行を仕込む。今回のような事故の再発防止になる。
