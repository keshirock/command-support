# command-support

## 西消防署 災害活動支援（災害別部隊活動方針）アプリ（PWA）

神戸市消防局「災害別部隊活動方針」（火災編24種別・災害編15種別・危機事象発生時の対応編）を、災害種別ごとの指揮カード（チェックリスト）としてアプリ化したものです。出動指令の種別に応じたカード（一般地区建物火災／交通事故／毒劇物災害 等）を選び、「活動段階」（出動指令→現場到着→活動中→収束・引揚げ）を切り替えると、状況把握・活動・安全管理・部隊整理の各フェーズで実施すべきタスクと未実施件数が一目でわかります。出動基準・屋内進入統制・特殊災害ゾーニング・水難救助要件などの資料参照、全文検索、オフライン動作に対応。

- 公開URL: https://keshirock.github.io/command-support/saigai/
- ファイル一式: `saigai/` ディレクトリ
- 注意: 正本は紙の資料です。特殊災害協力機関の連絡先等、資料に記載のない情報は掲載していません。

## 西消防署 震災タスク（震災初動対応）アプリ（PWA）

「令和７年度 震災初動対応マニュアル（神戸市西消防署）」を、役割別（参集職員・指揮活動班・情報班・現場出動隊・消防団支援班・後方支援班）のタスクチェックリストとしてアプリ化したものです。「発災からの経過」（発災直後→〜１時間→１〜４時間→４〜24時間→24〜72時間）を切り替えると、各フェーズで実施すべきタスクと未実施件数が一目でわかります。組織・部隊編成・活動方針・津波対策・無線・報告要領などの資料参照、全文検索、オフライン動作に対応。

- 公開URL: https://keshirock.github.io/command-support/shinsai/
- ファイル一式: `shinsai/` ディレクトリ
- 注意: 正本は紙のマニュアルです。非公開情報（携帯電話開局番号・メールアドレス等）は掲載していません。

### 共通機能（水防タスク・震災タスク・災害活動支援）

- ライト／ダークモード切替（右上の🌙／☀️ボタン。設定は両アプリ共通で端末に保存）
- 実施記録の出力：コピー／Word（.doc）／Excel（.csv）
- iPhoneのノッチ・ダイナミックアイランドを避けたセーフエリア表示

## 西消防署 水防タスク（風水害対応）アプリ（PWA）

「令和８年度 水防計画書（案）西消防署」を、役割別（指揮活動班・情報班・現場出動隊・消防団支援班・後方支援班）のタスクチェックリストとしてアプリ化したものです。「現在の体制」（平常時→待機→警戒→１号→２号→３号→解除・収束）を切り替えると、いま実施すべきタスクと未実施件数が一目でわかります。チェックすると実施時刻が記録され、時系列の実施記録（クロノロジー）としてコピーもできます。判断基準・様式・連絡先（タップ発信）などの資料参照、全文検索、オフライン動作に対応。チェック状態は端末内（localStorage）のみに保存されます。

- 公開URL: https://keshirock.github.io/command-support/suibou/
- ファイル一式: `suibou/` ディレクトリ（`index.html` 単一ファイル構成＋PWA用ファイル）
- 注意: 正本は紙の計画書です。文書内で「部外秘」とされている情報（消防団役員一覧・防コミ代表者一覧・神戸電鉄連絡先など）は掲載していません。

### セキュリティ対策

- 外部リソースの読み込みなし（完全自己完結）＋ Content-Security-Policy で外部スクリプト・接続をブロック
- 検索結果表示はHTMLエスケープ済み（XSS対策、攻撃文字列で注入テスト済み）
- `noindex` により検索エンジンにインデックスされない設定（URLを知る人のみアクセス）
- リファラ送信なし（外部リンクを開いてもアプリのURLが相手サイトに渡らない）
- Service Worker は同一オリジンのGETリクエストのみ処理
- ※ GitHub Pages は公開ホスティングのため、URLを知っていれば誰でも閲覧できます。掲載内容は公開可能な情報に限定していますが、より厳密なアクセス制限が必要な場合は認証付きホスティングへの移行を検討してください。

## あみだくじ班分け（PWA）

少人数ミーティング向けの「あみだくじ班分け」アプリです。GitHub Pages で公開し、スマホのホーム画面に追加するとオフラインでも使えます。

- 公開URL: https://keshirock.github.io/command-support/

### スマホでオフラインで使う手順

1. スマホのブラウザで上記の公開URLを開く（初回はネット接続が必要）
2. ホーム画面に追加する
   - **iPhone (Safari)**: 共有ボタン（□に↑）→「ホーム画面に追加」
   - **Android (Chrome)**: メニュー（︙）→「ホーム画面に追加」または「アプリをインストール」
3. 以降はホーム画面のアイコンから起動でき、機内モードなどオフラインでも動作します

### 構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | アプリ本体（単一ファイル） |
| `manifest.webmanifest` | PWA マニフェスト（ホーム画面追加用） |
| `sw.js` | Service Worker（オフラインキャッシュ） |
| `icon-192.png` / `icon-512.png` / `apple-touch-icon.png` | アプリアイコン |

※ このリポジトリは GitHub Pages（main ブランチから配信）が有効なので、main にマージすると自動で公開されます。
